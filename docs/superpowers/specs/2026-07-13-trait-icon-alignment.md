# 特质配图与文字对齐设计

## 范围

网站为地区特质和利益集团特质显示游戏内对应图标。覆盖州地区详情的“地区特质效果”、利益集团基础特质、国家详情中因风味替换而显示的利益集团特质，以及独立利益集团特质详情。筛选标签与紧凑概念标签继续只显示文字。

## 图标来源与映射

地区特质从 `site/assets/state-traits/<特质键>.png` 读取；利益集团特质从 `site/assets/interest-group-traits/<特质键>.png` 读取。构建时将游戏目录 `gfx/interface/icons/state_trait_icons` 与 `gfx/interface/icons/ig_trait_icons` 的 PNG 文件复制到上述网站资源目录。特质键与图标文件基名一致。

渲染函数以特质键生成图片路径和替代文字。图片加载失败时隐藏图片本身，文字卡片保持原有内容和间距，不显示破损图标。

## 版式

完整特质卡片采用固定 22 像素图标列与自适应文字列。图标容器从卡片首行起定位，并以 1.35 倍正文行高的视觉中线和标题对齐。标题、支持度、效果与说明位于文字列；文字换行时图标列不收缩，后续行均从同一文字起始线开始。地区特质效果卡片和利益集团特质卡片使用同一图标列规则。

风味利益集团沿用既有 `interestGroupTraitDetailsHtml` 与 `interestGroupTraitDetailCard` 渲染链，因此只要调用这套卡片函数，基础特质与替换后的特质都会取得相同的图标与对齐效果。

## 验证

新增静态检查，确认资源映射、卡片图标标记、风味渲染链和样式的固定图标列均存在。验证脚本还检查全部已引用的特质图标确实位于网站资源目录。浏览器分别检查含风味利益集团的国家详情与包含地区特质的州地区详情，确认无资源请求失败、图标与首行文字对齐、窄视口无横向溢出。
