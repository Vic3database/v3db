# 国家板块地图边界层级

## 目的

国家板块按开局国家归属着色时，国界需要高于州地区界显示。国境内部仍保留州地区界，作为较弱的辅助信息。地区、文化和公司板块继续使用现有的州地区边界表现。

## 现状与原因

`site/app.js` 的 `addMapBorders` 在一次像素遍历中同时判断州地区索引和开局归属索引。相邻像素同时属于不同州地区和不同国家时，当前颜色选择优先使用州地区界，因此国界无法覆盖在上层。

## 设计

国家板块使用两遍边界绘制。

第一遍仅绘制州地区界与海岸线。州地区界使用比当前更淡的颜色，保留一像素宽度；海岸线继续使用现有海域边界颜色。

第二遍仅在相邻陆地区域的开局归属国家不同处绘制国界。国界使用更深、更醒目的颜色，并覆盖第一遍中重合的州地区界。国界的像素范围应比州地区界更完整，避免在相邻州地区交界处出现断线。

非国家板块继续调用单层边界绘制，不改变颜色、边界判断或海岸线。地图数据、州地区索引和开局归属索引不需要重新生成。

## 验证

新增或扩展地图检查脚本，验证国家板块调用双层绘制，州地区边界先于国界绘制，国界判断依赖相邻像素的开局归属索引差异，非国家板块保留单层调用。

浏览器打开国家板块，选择含有跨国相邻州地区的区域，确认国界连续且明显高于内部州地区界；切换到地区板块，确认州地区界与改动前一致。检查页面控制台无错误。
