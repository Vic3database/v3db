# Vicdata WebP 图片资源优化设计

## 目标与范围

降低普通展示图片的首次下载与后续详情页加载体积，同时保持现有图片尺寸、透明通道和页面视觉结构。首批处理 `site/assets/map/flatmap_votp.png`，以及 `buildings`、`companies`、`laws`、`ideologies` 目录中的 PNG 图片。地图展示底图使用 WebP 质量 80；其余四类资料配图使用质量 85。

`site/assets/map/provinces.png` 保持 PNG。该文件承载地图省份的像素取色与交互定位，不能替换。国旗、小型特质图标、首页图标、品牌图标和 DLC 图标均不在本轮范围内。

## 生成与引用

新增一个可重复执行的 Node.js 转换脚本。脚本从指定 PNG 输入生成同目录同名的 `.webp` 文件，不改变原 PNG；运行结果输出每个目录的源文件数、转换成功数、原始总字节数和 WebP 总字节数。脚本在目标文件已存在且源文件未更新时跳过生成，支持在素材同步后增量运行。

普通展示图片通过 `picture` 元素提供 WebP 优先与 PNG 回退：支持 WebP 的浏览器请求 `.webp`，不支持时继续请求原有 `.png`。动态路径生成函数集中处理这项选择，避免在建筑、公司、法律和意识形态的多个渲染位置分别拼接格式。地图展示底图也采用相同的优先与回退方式；省份取色图仍直接使用原有 PNG 路径。

原 PNG 在首轮发布中保留。这样可以验证图片质量、透明边缘、动态引用与旧浏览器回退。通过浏览器与脚本验收后，再另行决定是否从生产发布包排除对应 PNG 回退文件。

## 验收

新增检查脚本验证：每个范围内的 PNG 都存在对应 WebP；WebP 的宽高与 PNG 相同；省份取色图仍为 PNG；前端对转换范围使用 WebP 优先与 PNG 回退；未处理目录不产生意外的 WebP 文件。转换脚本还要在运行结束后给出压缩前后体积与节省比例。

浏览器验收覆盖首页地图、国家详情中的建筑图、公司详情、法律详情与意识形态详情，确认图片未加载失败、未出现明显透明边缘异常，并记录相关资源的实际请求格式。发布前复查 Nginx 对 `.webp` 的长期缓存和 MIME 类型响应。

## 2026-07-14 转换结果

共生成 584 个 WebP 文件，且检查确认每个文件与对应 PNG 的宽高一致。转换范围的 PNG 合计 70,182,610 字节，WebP 合计 9,184,226 字节，单独请求 WebP 时减少 60,998,384 字节，约为 86.9%。其中地图展示底图从 15,160,427 字节降至 490,728 字节；省份取色图 `provinces.png` 未转换。

静态检查确认法律、建筑、公司与意识形态展示图均使用 WebP 优先的 `picture` 标记并保留 PNG 回退；国旗、利益集团、威望商品、DLC、首页、品牌和特质图标仍维持原格式。`check_publish_bundle.mjs` 会同时检查转换范围的 WebP 与 PNG，Nginx 现有静态资源规则已包含 `.webp`。
