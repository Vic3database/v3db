# 公司板块默认列表与详情定位图设计

## 目标

公司板块的默认路由 `#/company` 以筛选栏和完整公司列表为主体，不显示主地图。打开 `#/company/<公司键>` 后，在右侧详情的“位置”小节中显示一张小型定位图，帮助识别该公司关联的地点。

## 页面与交互

公司列表页隐藏既有的主地图面板，地图工具栏、模式选择器和地图画布均不占用页面空间。筛选栏、结果列表与现有排序和搜索行为保持不变。公司行单击仍保留当前选中行为，详情按钮仍进入带公司键的详情路由。

公司详情页保留现有“基础”“经营”“条件脚本”内容，并新增“位置”小节。“位置”小节置于“基础”之后，显示一张小型、只读地图及地点说明。该地图不包含全局筛选、地图模式选择、缩放重置按钮、拖拽、滚轮缩放或地图点选；其作用仅限于显示本公司的关联范围。

## 地点推导

通用公司不显示定位图，也不显示“位置”小节，即使条件脚本中出现战略区域。彩蛋公司不纳入本次新增的定位图。

史实公司显示“位置”小节。用于地图高亮和视野缩放的州地区按以下顺序合并并去重：`preferred_headquarters`、`referenced_state_regions`、`referenced_strategic_regions` 展开的州地区、`referenced_geographic_regions` 展开的州地区。地图沿用公司地图的颜色层与边界绘制逻辑，只突出当前公司的关联州地区。

罗马尼亚铁路 `company_cfr` 和奥斯曼烟草公司 `company_ottoman_tobacco_regie` 没有上述地点字段，使用定向补充规则：前者取罗马尼亚文化 `romanian` 的本土州地区，后者取土耳其文化 `turkish` 的本土州地区。文化本土推导只适用于这两个公司，不能把其他公司的一般文化条件自动视为经营地点。

若今后的史实公司在上述常规字段和定向补充规则中均无可定位州地区，详情显示“暂无可定位地点”，不创建空白地图画布。

## 验收条件

`#/company` 不渲染主地图，页面默认展示完整公司列表。通用公司详情不出现定位图。具备总部或关联州地区的史实公司显示并聚焦对应区域；仅具备战略区域或地理区域的史实公司在展开后的州地区范围内显示地图。罗马尼亚铁路和奥斯曼烟草公司的地图分别以罗马尼亚文化本土和土耳其文化本土为范围。现有公司详情字段、列表筛选、排序、搜索与路由不得回归。
