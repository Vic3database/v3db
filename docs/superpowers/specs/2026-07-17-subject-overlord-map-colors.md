# 附属国宗主国颜色

## 目的

国家板块的开局地图新增“低级附属国显示为宗主国的颜色”设置，默认开启。适用的附属国使用直接宗主国的标准色，便于识别开局政治控制范围；受保护国和朝贡国继续使用自身颜色。

左侧国家搜索框输入名称或 Tag 后，所有搜索命中的国家使用自身颜色。通过全局搜索打开国家时，该国家使用自身颜色。普通国家列表点击不改变附属国颜色。

## 数据

抽取脚本读取 `common/history/diplomacy/00_subject_relationships.txt`。文件中外层 `c:TAG` 是宗主国，内部 `create_diplomatic_pact` 的 `country = c:TAG` 是附属国，`type` 是开局附属关系类型。

为国家数据增加以下字段：

- `starting_overlord_tag`：直接宗主国 Tag；没有时为空。
- `starting_subject_type`：开局附属关系类型；没有时为空。
- `starting_subject_uses_overlord_color`：是否属于地图借色范围。

借色范围包括 `puppet`、`vassal`、`crown_land`、`chartered_company`、`colony`、`dominion` 和 `personal_union`。`protectorate` 与 `tributary` 明确排除。国家详情和前端运行时数据保留宗主国的 Tag、名称与类型，供颜色计算和资料展示使用。

已核对的开局样本为：东印度公司 `BIC` 隶属英国 `GBR`，类型 `chartered_company`；西藏 `TIB` 隶属清 `CHI`，类型 `vassal`；的黎波里塔尼亚 `TRI` 隶属奥斯曼 `TUR`，类型 `puppet`；匈牙利 `HUN` 隶属奥地利 `AUS`，类型 `crown_land`。

## 前端行为

设置弹窗加入复选框“低级附属国显示为宗主国的颜色”，默认勾选，并通过独立本地存储键保存。关闭设置时，所有国家恢复自身颜色。

国家地图颜色按以下顺序决定：

1. 地图筛选的变灰规则优先于普通着色。
2. 左侧国家搜索框当前命中的国家使用自身颜色。
3. 通过全局搜索打开的国家使用自身颜色。
4. 设置开启且国家属于借色范围时，使用直接宗主国颜色。
5. 受保护国、朝贡国和其他国家使用自身颜色。

左侧搜索命中集合由当前搜索文本计算，不依赖列表选择。全局搜索打开国家时，路由或全局搜索状态记录该 Tag，离开该路径或重新执行全局搜索后清除。普通列表点击不写入该状态，因此不会使附属国恢复自身颜色。

## 验证

抽取检查验证四个样本的宗主国和类型，并验证受保护国、朝贡国的借色标记为否。网页数据检查验证新字段从数据库进入国家数据块。

前端检查验证默认开启、设置弹窗与本地保存、地图缓存签名、颜色优先级以及搜索命中和全局搜索 Tag 的处理。浏览器检查设置开关、`BIC`、`TIB`、`TRI`、`HUN` 的默认借色，受保护国和朝贡国的自身颜色，左侧搜索恢复自身颜色，以及全局搜索打开国家后的自身颜色。检查控制台无错误。
